概览

容器主要包括Collection和Map两种，Collection存储着对象的集合，而Map存储着键值对（两个对象）的映射表。

Collection

set list queue

set

  • 无需，不可重复
  • hashSet:基于哈希表实现（元素是链表的数组），支持快速查找，但不支持有序性操作。并且失去了元素的插入顺序信息，也就是说使用iterator遍历HashSet得到的结果是不确定的。

哈希表依赖于哈希值存储。底层依赖于HashCode()和equals（）方法相同则不添加。

子类LinkedHashSet底层哈希表（唯一性）链表（有序）

  • TreeSet：基于红黑树实现，支持有序性操作，例如根据一个范围查找元素的操作。但是查找效率不如HashSet，HashSet查找的时间复杂度为O(1)，TreeSet则为O(logN)。

自然排序和比较器排序。来确保唯一性。实现Comparable接口， comparator

  • LinkedHashSet：具有HashSet的查找效率，且内部使用双向链表维护元素的插入顺序。

    list

  • ArrayList:基于动态数组实现，支持随机访问。查找快，增删慢，线程不安去，效率高
  • LinkedList：基于双向链表实现，只能顺序访问，但是可以快速地在链表中间插入和删除元素。不仅如此，LinkedList还可以用作栈、队列和双向队列。

查找慢，增删快，线程不安全，效率高
-Vector:和ArrayList类似，但它是安全的。

queue

  • LinkedList：可以用他来实现双向队列。
  • PriorityQueue:基于堆结构实现，可以用它来实现优先队列。

    Map

    将键映射到值的对象，一个映射不能包含重复的键。
  • TreeMap:基于红黑树实现。
  • hashMap:基于哈希表实现。
  • hashTable:和hashMap类似，但它是线程安全的，这意味着同一时刻多个线程可以同时写入hashTable并且不会导致数据不一致，它是遗留问题，不应该去使用他，现在可以使用ConcurrentHashMap来支持线程安全，并且ConcurrnetHashMap的效率更高，因为引入了分段锁。
  • linkedHashMap:使用双向蓝标来维护元素的顺序，顺序为插入顺序或者最近最少使用顺序。

    容器中的设计模式

    迭代器模式

    collection继承了iterable接口，其中iterator（）方法能够产生一个Iterator对象，通过这个队形就能迭代遍历collection中的元素。

从jdk1.5之后就可以使用foreach方法来遍历实现了iterable接口的聚合对象。

适配器模式

java.util.Arrays#asList()可以把数组类型转换为list类型。

应该注意的是aslist（）的参数为泛型的边长参数，不能使用基本数据类型数组作为参数，只能使用相应的包装类型数组。

CopyOnWriteArrayList

读写分离

写操作在一个复制的数组上进行，读操作还是在原始数组中进行，读写分离，互不影响。

写操作需要加锁，防止并发写入时导致写入数据丢失。

写操作结束之后需要把原始数组指向新的复制数组。

适用场景

CopyOnWriteArrayList 在写操作的同时允许读操作，大大提高了读操作的性能，因此很适合读多写少的应用场景。

但是 CopyOnWriteArrayList 有其缺陷：

  • 内存占用：在写操作时需要复制一个新的数组，使得内存占用为原来的两倍左右；
  • 数据不一致：读操作不能读取实时性的数据，因为部分写操作的数据还未同步到读数组中。

所以 CopyOnWriteArrayList 不适合内存敏感以及对实时性要求很高的场景。
